本地文件夹，在git bash或cmd中使用命令git init初始化为git仓库。

此时，文件在未暂存区，已暂存文件代表暂存区，选中未暂存文件，并暂存文件，文件从未暂存区——移到——暂存区。
若要将文件提交至版本库，1.在自己的区域编写提交记录； 2.点击提交；因此文件从暂存区——移到——版本库。

二.创建两个及以上文件后

此时Git分区的作用就体现了出来。

思考：试想一下为什么要在未暂存区和版本库之间要有一个暂存区？

如果中间没有暂存区，那么在未暂存区无论是否准备完全，都要提交到版本库中；但如果有了暂存区，就可以在暂存区进行进一步的筛选检查。

三.命令操作

1.未暂存区–>暂存区 git add 或 git add * ；

2.暂存区–>版本库 git commit m”描述信息” ；

3.软合并（版本库–>暂存区） git reset –mix 版本号 ；
强行合并（丢掉此次提交之后的所有提交） 不要了 ；
